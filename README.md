# 汕头大学校园网自动认证守护进程

汕头大学校园网自动认证守护进程，用于解决午夜清除 cookie 导致的断网问题。程序会定时检测网络状态，当检测到网络断开时自动重新认证。

## 功能特点

- 自动检测网络连通性
- 网络断开时自动重新认证
- 支持汕头大学锐捷认证系统
- 定时检测（默认 60 秒）
- 实时日志输出

## 环境要求

- Python 3.6+
- requests 库

## 安装依赖

```bash
pip install requests
```

## 配置说明

1. 编辑 `stu-network.conf` 配置文件：
   - 第一行：校园网账号（学号）
   - 第二行：校园网密码

示例：
```
2023123456
mypassword123
```

## 使用方法

### 方法一：直接运行 Python 脚本

```bash
python StuNetworkDaemon.py stu-network.conf
```

### 方法二：使用批处理文件（Windows）

双击运行 `run_stu_daemon.bat`

## 程序说明

### 工作原理

1. 程序启动后，每隔 60 秒检测一次网络状态
2. 通过 ping 汕头大学 DNS (202.192.240.87) 和阿里 DNS (223.5.5.5) 判断网络连通性
3. 当检测到网络断开时，自动向认证服务器发起认证请求
4. 认证成功后继续监控，失败则等待下次检测

### 认证流程

1. 访问测试页面 (http://1.1.1.2) 获取认证跳转链接
2. 提取认证页面 URL 和相关参数
3. 发送 POST 请求到认证接口完成登录
4. 认证结果通过日志输出

### 日志输出

程序会输出以下信息：
- 启动信息（用户名、检测间隔）
- 网络状态检测结果
- 认证尝试和结果
- 错误信息

示例输出：
```
[2024-01-15 10:30:00] 汕头大学校园网自动认证守护进程启动
[2024-01-15 10:30:00] 用户名：2023123456
[2024-01-15 10:30:00] 检测间隔：60 秒
[2024-01-15 10:31:00] 网络正常
[2024-01-15 10:32:00] 网络断开，尝试重新认证
[2024-01-15 10:32:01] 认证成功：登录成功
[2024-01-15 10:32:01] 认证成功，网络已恢复
```

## 停止程序

- Windows：按 `Ctrl+C` 终止程序
- Linux/Mac：按 `Ctrl+C` 发送 SIGINT 信号

## 自定义配置

如需修改检测间隔，可编辑 `StuNetworkDaemon.py` 中的 `check_interval` 参数：

```python
# 认证间隔（秒）
self.check_interval = 60  # 修改此值
```

## 常见问题

### 1. 提示找不到 Python

请确保已正确安装 Python 3.6+，并将 Python 添加到系统环境变量 PATH 中。

### 2. 认证失败

- 检查账号密码是否正确
- 确认网络连接正常
- 查看日志输出中的具体错误信息

### 3. 程序运行后无法上网

- 检查是否成功认证（查看日志）
- 确认认证服务器地址是否正确
- 尝试手动访问认证页面测试

## 文件说明

- `StuNetworkDaemon.py` - 主程序文件
- `stu-network.conf` - 配置文件（需自行填写账号密码）
- `run_stu_daemon.bat` - Windows 启动脚本

## 注意事项

1. 请将账号密码信息安全保存，不要泄露配置文件
2. 程序需要持续运行，建议使用后台服务或任务计划程序
3. 首次运行前请确保已连接到汕头大学校园网
